Plugin para exportar em CSV

27-03-2016 Deixe um comentário Plugin

Beleza pessoal? Bom em primeiro lugar gostaria de pedir desculpas pelo tempo que passei sumido (realmente foi mais de um mês rs) e dizer que houve muitos projetos desde o último post e é sobre um desses projetos que venho falar hoje, um plugin que exporta de JSON para CSV criado por mim com o apoio de dois amigos no trabalho.

Por que plugin?

Bom pessoal em primeiro lugar gostaria de falar o porque o plugin se tornou realmente necessário e porque transformar códigos em plugins realmente é uma boa prática, a necessidade surgiu quando em um projeto tive que desenvolver um algoritmo que pegava um JSON e exportava para CSV, até ai tudo bem, o que eu precisava era de uma única função, mais como boa prática fui aprimorando código e em seguida tive que coloca-lo em outro projeto já fazendo mais alguns modificações e nesse ponto descobri que exportar documentos CSV é uma prática muito comum em sistemas grandes hoje e logo não poderia sair duplicando mais ainda meu algoritmo.

A partir desse ponto busquei boas práticas na construção de um plugin puro e genuianamete Javascript já que até o momento tinha evitado a dependencia de Jquery embora o projeto já estivesse com a biblioteca disponível isso iria me causar uma depencia externa, algo que estava evitando, e como veremos a seguir o plugin me permite uma versatilidade já que tenho que apenas incluir os arquivo e instancio-lo para poder usá-lo

Sobre o plugin

Quando se busca criar um plugin acaba encontrando algumas diretivas de boas práticas que quando seguidas se trazem beneficios e causa uma qualidade maior e uma compreensão no uso do código, sendo o princípio SOLID um dos mais difundidos

S RP - ingle Responsibility (Quando resolvi criar o plugin, sabia que ele só tinha um objetivo).

O CP - Capaz de estender um comportamento sem modificá-lo.

L SP - As classes derivadas devem ser substituíveis por suas classes base.

I SP - Muitas interfaces específicas são melhores do que uma interface única.

D IP - Dependa de uma abstração e não de uma implementação.

Além dos principios SOLID

Uma boa prática é trabalhar com IIFE para criar um ambiente fechado para o seu módulo. Closures pode ser aproveitado para criar um ambiente privado, onde você tem controle sobre os dados que você irá disponibilizar.

E vamos ao plugin

Plugin Javascript que converte um JSON em um arquivo CSV

Criado exclusivamente para converter um JSON em um arquivo CSV
